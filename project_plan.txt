PROGRAM OVERVIEW

This program is designed to simulate a real-world technology company environment in which students work as cross-functional product teams. Over a period of six months, students will design, develop, deploy, and present a production-grade software system that integrates mobile applications, web applications, artificial intelligence orchestration, and quantum computing algorithms.

PROJECT MANDATE

You are assigned to design, develop, deploy, and document a production-grade software product that integrates mobile applications, a web application, artificial intelligence orchestration, and quantum computing algorithms.

This assignment must be treated as an industry-level product development project. The final system must be functional, deployable on the cloud, and supported with full technical documentation.

The system must consist of:

Android mobile application

iOS mobile application

Web and desktop application

Backend server and database

AI orchestration service

Quantum algorithm module

DevOps and cloud deployment pipeline

PROJECT DOMAIN SELECTION

One of the following domains must be selected and approved before development begins. All platforms and components must be built for the selected domain.

Domain Option 1: Quantum Traffic Optimization
A system that optimizes traffic flow using AI predictions and quantum optimization algorithms.

Domain Option 2: Quantum-Assisted Drug Discovery
A platform that combines AI models and quantum simulations to analyze molecular structures.

Domain Option 3: Smart Energy Grid Optimization
A system that balances renewable energy supply and demand using AI forecasting and quantum optimization.

Domain Option 4: Financial Risk and Portfolio Optimization
A platform using AI forecasting and quantum probability modeling for financial risk analysis.

Domain Option 5: Climate Modeling and Disaster Prediction
A system that integrates AI climate analysis with quantum simulations for extreme weather prediction.

REQUIRED SYSTEM COMPONENTS

3.1 Mobile Applications
Two native-style mobile applications must be developed:

Android version following Material Design principles

iOS version following Apple Human Interface Guidelines

Both applications must support authentication, dashboards, data visualization, and result display from AI and quantum modules.

3.2 Web and Desktop Application
A responsive web application must be developed for desktop and laptop usage. It must provide full system functionality, administrative access, and advanced analytics dashboards.

3.3 Backend System
A backend server must be developed to handle:

User authentication and authorization

API endpoints for all system features

Communication with AI and quantum modules

Database operations

Logging and error handling

Recommended technologies include Node.js with Express.js and REST or GraphQL APIs.

3.4 Database Layer
A structured database must be designed and implemented. Acceptable options include PostgreSQL or MongoDB. Redis may be used for caching and performance improvements.

3.5 AI Orchestration Service
An AI service must be developed to:

Process system input data

Run predictive or analytical models

Determine when classical processing is sufficient

Determine when to invoke the quantum module

Return optimized outputs to the backend

The AI service may be developed using Python with suitable machine learning libraries.

3.6 Quantum Algorithm Module
A quantum computing component must be designed and implemented. This module must:

Address a computational challenge within the selected project domain

Be implemented using a framework such as Qiskit, Cirq, or PennyLane

Include quantum circuit design and simulation

Be integrated with the AI orchestration service

Provide performance comparison with a classical approach

DEVELOPMENT PHASES AND TIMELINE

PHASE 1: PRODUCT DEFINITION AND UI/UX DESIGN (Weeks 1–4)

Define system features and user flows. Produce complete UI/UX designs in Figma for Android, iOS, and Web. Establish a design system and interactive prototype.

Deliverables
Approved feature list
User flow diagrams
High-fidelity designs
Clickable prototype

PHASE 2: FRONTEND IMPLEMENTATION (Weeks 5–8)

Set up project repositories and implement the user interface for all platforms. Develop authentication flows, dashboards, and core feature screens.

Deliverables
Functional frontend for Android, iOS, and Web
Reusable UI components

PHASE 3: BACKEND AND DATABASE DEVELOPMENT (Weeks 9–12)

Develop server infrastructure, database schemas, authentication systems, and APIs. Integrate frontend applications with backend services.

Deliverables
Working backend APIs
Database with structured schema
API documentation

PHASE 4: AI ORCHESTRATION AND QUANTUM DEVELOPMENT (Weeks 13–16)

Develop the AI orchestration service and integrate machine learning models. Design and simulate the quantum algorithm. Connect the AI system with the quantum module and backend.

Deliverables
AI orchestration service
Quantum algorithm implementation
Integration between AI and quantum systems
Technical documentation of algorithms

PHASE 5: DEVOPS AND CLOUD DEPLOYMENT (Weeks 17–20)

Containerize all services, set up CI/CD pipelines, and deploy the system on a cloud platform. Configure monitoring and logging.

Deliverables
Cloud-deployed backend, frontend, AI, and database
CI/CD pipeline configuration
Deployment documentation

PHASE 6: TESTING AND FINAL RELEASE (Weeks 21–24)

Conduct system-wide testing, including unit tests, API tests, performance tests, and basic security validation. Fix issues and optimize performance. Prepare final documentation and demonstration.

Deliverables
Test reports
Optimized production system
Final technical documentation
Demonstration of working system
